Key Differences from Feature Development:

  • QA/Bug Reporter Role: Much more active throughout the process - responsible for verification and test case creation
  • Emergency Reviewers: Special role activated only for P1 bugs with 2-hour SLA
  • Support Team: Engaged for P1 bugs to verify customer impact and coordinate fixes
  • Faster Timeline: 1-3 days vs 2-5 days for features, with P1 bugs getting expedited treatment
  • Verification Focus: Heavy emphasis on confirming the bug is actually fixed in production
  • Branch Cleanup: Bug fix branches are automatically deleted after merge (not optional)

Severity-Based Variations:

  • P1 (Critical): Emergency reviewers, 2-hour review SLA, 4-hour monitoring, support team engagement
  • P2 (High): Standard review process, 24-hour monitoring, normal timeline
  • P3 (Low): Standard review process, 24-hour monitoring, may be batched with other fixes
